    Getting Started

    Environment Setup

    1. Prerequisites

    • Python 3.11 (strict requirement)
    • PostgreSQL 13+
    • Redis 6+
    • Git

    2. Installation

    # Clone repository
git clone https://github.com/PavelBelove/karhuno.git
cd karhuno

# Create virtual environment
python3.11 -m venv venv
source venv/bin/activate  # or venv\Scripts\activate on Windows

# Upgrade pip and install wheel
pip install --upgrade pip wheel setuptools

# Install dependencies from frozen requirements
pip install -r requirements.frozen.txt

    3. Configuration

    # Copy example environment file
cp .env.example .env

# Generate secret key
python -c "import secrets; print(secrets.token_hex(32))"

    Edit .env file: 

    SECRET_KEY=your_generated_key
DATABASE_URL=postgresql://user:password@localhost:5432/karhuno
REDIS_URL=redis://localhost:6379/0
XMLSTOCK_API_KEY=your_api_key
OPENAI_API_KEY=your_api_key

    Development Scripts

    Start Development Environment

    # start.sh
#!/bin/bash

# Start Redis if not running
redis-server --daemonize yes

# Start Celery worker
celery -A app.celery worker --loglevel=info &
echo $! > celery.pid

# Start Flask development server
flask run --debug

    Stop Development Environment

    # stop.sh
#!/bin/bash

# Kill Celery workers (sometimes requires force)
if [ -f celery.pid ]; then
    pid=$(cat celery.pid)
    kill -9 $pid
    rm celery.pid
fi

# Stop Redis
redis-cli shutdown

# Note: Flask server stops with Ctrl+C

    Usage: 

    # Start development environment
./start.sh

# Stop development environment
./stop.sh

    Development Tools

    1. Testing

    # Install development tools
pip install -r requirements.dev.txt

# Run tests
pytest

# Run with coverage
pytest --cov=app tests/

    2. Code Quality

    # Run linting
flake8 app tests

# Run type checking
mypy app

    Development Workflow

    1. Code Style

    • Follow PEP 8 guidelines
    • Use type hints
    • Write docstrings in Google format
    • Keep functions focused and under 50 lines

    2. Git Workflow

    # Create feature branch
git checkout -b feature/your-feature

# Make changes and commit
git add .
git commit -m "feat: description"

# Push changes
git push origin feature/your-feature

    3. Testing

    # Run specific test file
pytest tests/test_search_tasks.py

# Run with verbose output
pytest -v

# Run tests matching pattern
pytest -k "test_search"
